# 05 - 模板系统设计

## 设计目标

提供预置经典工作流模板 + 支持用户自定义模板，两者结合覆盖主要使用场景。

## 模板存储

### 内置模板

路径：`src/templates/`

| 模板文件 | 批次 | 说明 | 可参数化字段 |
|----------|------|------|-------------|
| `txt2img.json` | 第一批 MVP | 标准文生图 | checkpoint, prompt(+/-), width, height, steps, cfg, sampler, scheduler, seed, batch_size |
| `img2img.json` | 第一批 MVP | 图生图 | 同上 + input_image, denoise |
| `inpaint.json` | 第二批 | 局部重绘 | 同上 + mask_image |
| `upscale.json` | 第二批 | 超分放大 | input_image, upscale_model, scale_factor |
| `animation.json` | 第二批 | 动画生成 | 基于 AnimateDiff/SVD，帧数、帧率、运动强度等 |

### 用户自定义模板

路径：`user_templates/`

用户可以将 ComfyUI 导出的 `.json` 工作流文件放入此目录，MCP Server 启动时自动扫描并暴露给 Agent。

## 模板参数化机制

### 内置模板参数化

内置模板使用**占位符**标记可替换参数：

```json
{
  "3": {
    "inputs": {
      "seed": "{{seed}}",
      "steps": "{{steps}}",
      "cfg": "{{cfg}}",
      "sampler_name": "{{sampler_name}}",
      "scheduler": "{{scheduler}}",
      "denoise": "{{denoise}}",
      "model": ["1", 0],
      "positive": ["2", 0],
      "negative": ["2", 1],
      "latent_image": ["4", 0]
    },
    "class_type": "KSampler"
  }
}
```

Agent 调用 `load_template("txt2img", params={...})` 时，Server 将占位符替换为实际值。

### 用户模板参数化

用户模板是完整的 ComfyUI JSON，没有占位符。Agent 加载后需要通过**原子操作**（`update_node`）来修改参数。

内置模板相当于"带参数化标记的快捷方式"，用户模板相当于"自定义完整工作流"。

## 模板加载流程

```
load_template("txt2img", {checkpoint: "xxx.safetensors", width: 1024, ...})
        │
        ▼
1. 查找模板文件
   ├── 内置模板 → src/templates/txt2img.json
   └── 用户模板 → user_templates/{name}.json
        │
        ▼
2. 如果是内置模板：
   ├── 读取模板 JSON
   ├── 合并用户提供的 params（覆盖默认值）
   ├── 替换所有 {{占位符}}
   └── 返回完整工作流
        │
3. 如果是用户模板：
   ├── 读取模板 JSON
   ├── 如果有 params，遍历所有节点尝试匹配更新
   └── 返回完整工作流
        │
        ▼
4. 存入当前工作流上下文（内存）
   Agent 可继续使用原子操作修改
```
