# 06 - 图片处理方案

## 设计目标

生成的图片需要返回给 Agent，同时兼顾预览体验和性能。

## 方案：缩略图 base64 + 原图路径

| 返回内容 | 用途 | 说明 |
|----------|------|------|
| 缩略图 base64 | Agent 预览 | 最大 512px，压缩后 base64 编码，直接嵌入返回值 |
| 原图路径 | 用户获取原图 | 本地文件系统路径，Agent 可通过文件工具读取 |

## 图片获取流程

```
execute_workflow() → 轮询 get_execution_status() → 完成
                                                      │
                                                      ▼
                                            get_execution_history()
                                                      │
                                                      ▼
                                              获取图片文件名
                                                      │
                                                      ▼
                                            get_generated_image(filename, thumbnail=true)
                                                      │
                                                      ▼
                                            ┌─────────────────────┐
                                            │ thumbnail_base64    │ ← Agent 可直接展示
                                            │ original_path       │ ← 用户需要原图时提供
                                            │ width, height, size │
                                            └─────────────────────┘
```

## 缩略图生成策略

1. 从 ComfyUI 下载原图到本地临时目录
2. 使用 PIL/Pillow 生成缩略图（等比缩放，最大边长 512px）
3. 缩略图编码为 base64
4. 返回 `thumbnail_base64` + `original_path`

## 图片上传流程

当需要 img2img / inpaint 等场景时，Agent 需要先将图片上传到 ComfyUI：

```
upload_image(local_path, subfolder="")
        │
        ▼
1. 读取本地图片文件
2. 通过 ComfyUI /upload/image 接口上传
3. 返回上传后的文件名和路径
        │
        ▼
4. Agent 在 build_workflow 的 IR 中引用该文件名
   {"input_image": "uploaded_image.png"}
```

## 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `THUMBNAIL_MAX_SIZE` | 512 | 缩略图最大边长(px) |
| `THUMBNAIL_FORMAT` | `"JPEG"` | 缩略图格式 |
| `THUMBNAIL_QUALITY` | 85 | 缩略图压缩质量(1-100) |
| `TEMP_DIR` | 系统临时目录 | 缩略图临时存储 |
