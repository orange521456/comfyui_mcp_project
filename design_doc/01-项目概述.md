# 01 - 项目概述

## 项目名称

comfyui_mcp_project

## 项目目标

通过 MCP（Model Context Protocol）将本地 ComfyUI 服务接入 AI Agent（如 Trae），使用户只需使用自然语言即可完成 ComfyUI 中的所有操作，包括：新建工作流、配置模型、通过工作流生成图片等。

## 目标用户场景

### 场景A（核心）：自然语言创建并执行工作流

用户说："帮我生成一张赛博朋克风格的城市夜景，用 SDXL 模型"

Agent 自动完成：选模型 → 搭节点 → 填参数 → 执行 → 返回图片。

### 场景B（核心）：基于已有工作流修改并执行

用户说："加载我的 `workflow_1.json`，把 checkpoint 换成 `dreamshaper_v8`，prompt 改成 xxx，然后执行"

Agent 自动完成：加载模板 → 修改参数 → 执行 → 返回图片。

### 场景C（后续扩展）：img2img / inpaint 等复杂工作流

用户说："创建一个 img2img 工作流，输入图片是 xxx，denoise 0.6，输出 1024x1024"

## 开发策略

### 工作流创建策略：模板驱动 + 动态构建结合

| 策略 | 做法 | 覆盖场景 |
|------|------|----------|
| 模板驱动 | 预置经典工作流模板（txt2img、img2img、inpaint 等），agent 选模板 + 填参数 | 80% 常见场景 |
| 动态构建 | agent 通过 MCP 工具逐个创建节点、连线，自由搭建工作流 | 兜底复杂场景 |

### 通信方式：REST + WebSocket 双通道

- REST API：提交工作流、查询历史、下载图片、管理队列
- WebSocket：实时推送执行事件（execution_start、progress、executing、executed、execution_error）
- 自动降级：WebSocket 连接失败时自动回退到 REST 轮询，保证可靠性

## 技术选型

| 项 | 选型 | 理由 |
|----|------|------|
| MCP Server 语言 | Python（FastMCP） | ComfyUI 生态以 Python 为主，集成自然 |
| ComfyUI 通信 | REST API + WebSocket | REST 负责提交/查询，WebSocket 负责实时进度推送 |
| 工作流表示 | 中间表示层（IR） | 屏蔽 ComfyUI 原生 JSON 复杂度 |
| 图片返回 | 缩略图 base64 + 原图路径 | agent 可预览，用户按需取原图 |
| 错误处理 | 统一错误格式 | 所有工具返回 `{"success": false, "error": "..."}` |
| 日志 | stderr + 文件 | 不污染 stdio（MCP 协议通道），按天轮转 |
| 只读数据 | MCP Resources | 模型列表、节点类型等暴露为 Resource，而非仅 Tool |
| ComfyUI 发现 | 启动时自动检测 | 检测 ComfyUI 是否在运行，不可用时给出明确提示 |
| 多客户端 | 暂不支持 | 单 Agent 连接，后续可升级 |
