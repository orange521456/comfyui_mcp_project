# 03 - 工作流中间表示（IR）

## 设计目标

屏蔽 ComfyUI 原生 JSON 的复杂度，提供一套声明式的 Pipeline 中间表示（Intermediate Representation），让 Agent 能够用自然语言描述的工作流意图，被准确翻译为可执行的 ComfyUI 节点图。

```
自然语言 → Agent理解 → Pipeline IR → module_engine → ComfyUI原生JSON
```

## IR 设计原则

1. **声明式**：用户只需描述"要做什么"，不需要关心节点 ID、连线索引
2. **模块化**：每个处理步骤是一个独立模块，引擎自动连线
3. **可组合**：模块可以任意组合，覆盖从简单到复杂的场景
4. **可扩展**：新增模块只需注册到引擎，不影响现有逻辑

## Pipeline IR 结构

### 完整结构

```python
{
    "type": "pipeline",
    "meta": {
        "positive_prompt": "a cyberpunk city at night",
        "negative_prompt": "bad quality, blurry",
        "width": 1024,
        "height": 1024,
        "steps": 20,
        "cfg": 7.0,
        "sampler_name": "euler",
        "scheduler": "normal",
        "seed": -1,
        "batch_size": 1,
        "filename_prefix": "ComfyUI"
    },
    "pipeline": [
        {"module": "load_checkpoint", "checkpoint": "sd_xl.safetensors"},
        {"module": "lora", "lora_name": "style.safetensors", "strength": 0.8},
        {"module": "prompt_pos"},
        {"module": "prompt_neg"},
        {"module": "controlnet", "image": "pose.png", "control_net_name": "openpose.safetensors"},
        {"module": "empty_latent"},
        {"module": "ksampler"},
        {"module": "vae_decode"},
        {"module": "upscale", "upscale_model": "4x-UltraSharp.pth"},
        {"module": "save_image"}
    ]
}
```

### meta 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `positive_prompt` | string | 是 | - | 正向提示词 |
| `negative_prompt` | string | 否 | `""` | 反向提示词 |
| `width` | int | 否 | 512 | 图片宽度 |
| `height` | int | 否 | 512 | 图片高度 |
| `steps` | int | 否 | 20 | 采样步数 |
| `cfg` | float | 否 | 7.0 | CFG scale |
| `sampler_name` | string | 否 | `"euler"` | 采样器名称 |
| `scheduler` | string | 否 | `"normal"` | 调度器 |
| `seed` | int | 否 | -1(随机) | 随机种子 |
| `batch_size` | int | 否 | 1 | 批量生成数量 |
| `filename_prefix` | string | 否 | `"ComfyUI"` | 输出文件名前缀 |

### pipeline 模块注册表

| 模块名 | 对应节点 | 输入依赖 | 输出类型 | 说明 |
|--------|---------|---------|---------|------|
| `load_checkpoint` | CheckpointLoaderSimple | - | MODEL, CLIP, VAE | 加载主模型 |
| `lora` | LoraLoader | MODEL, CLIP | MODEL, CLIP | 加载 LoRA，串联到模型链 |
| `prompt_pos` | CLIPTextEncode | CLIP | CONDITIONING(+) | 正向提示词编码 |
| `prompt_neg` | CLIPTextEncode | CLIP | CONDITIONING(-) | 反向提示词编码 |
| `empty_latent` | EmptyLatentImage | - | LATENT | 创建空 latent |
| `controlnet` | LoadImage + ControlNetLoader + ApplyControlNet | CONDITIONING(+), CONDITIONING(-) | CONDITIONING(+), CONDITIONING(-) | 复合模块，控制网增强 |
| `ip_adapter` | LoadImage + IPAdapterModelLoader + IPAdapterApply | MODEL | MODEL | 复合模块，IP适配器 |
| `ksampler` | KSampler | MODEL, CONDITIONING(+), CONDITIONING(-), LATENT | LATENT | 核心采样 |
| `vae_decode` | VAEDecode | LATENT, VAE | IMAGE | latent→图像 |
| `upscale` | UpscaleModelLoader + ImageUpscaleWithModel | IMAGE | IMAGE | 复合模块，超分放大 |
| `save_image` | SaveImage | IMAGE | - | 保存输出 |

## 自动连线机制

每个模块声明自己的输入需求和输出类型。引擎按 pipeline 顺序处理，自动查找最近的生产者进行连线：

```
load_checkpoint → 产出 MODEL, CLIP, VAE
lora           → 消费 MODEL, CLIP → 产出 MODEL, CLIP（替换）
prompt_pos     → 消费 CLIP → 产出 CONDITIONING(+)
prompt_neg     → 消费 CLIP → 产出 CONDITIONING(-)
controlnet     → 消费 CONDITIONING(+/-) → 产出 CONDITIONING(+/-)（增强）
empty_latent   → 产出 LATENT
ksampler       → 消费 MODEL, CONDITIONING(+/-), LATENT → 产出 LATENT
vae_decode     → 消费 LATENT, VAE → 产出 IMAGE
upscale        → 消费 IMAGE → 产出 IMAGE
save_image     → 消费 IMAGE
```

连线规则：
- 每个模块声明它需要的输入类型（如 `MODEL`, `CLIP`, `LATENT`）
- 引擎向前查找最近一个产出该类型的模块，自动建立连线
- 如果找不到所需类型的生产者，报错

## 典型 Pipeline 示例

### 简单文生图

```python
{
    "type": "pipeline",
    "meta": {"positive_prompt": "a cat", "checkpoint": "sd_xl.safetensors"},
    "pipeline": [
        {"module": "load_checkpoint", "checkpoint": "sd_xl.safetensors"},
        {"module": "prompt_pos"},
        {"module": "prompt_neg"},
        {"module": "empty_latent"},
        {"module": "ksampler"},
        {"module": "vae_decode"},
        {"module": "save_image"}
    ]
}
```

## 向后兼容：扁平 IR（已废弃）

原有的扁平 IR（`type: "txt2img"` / `type: "img2img"`）保留兼容，但新功能不再扩展。Agent 应优先使用 Pipeline IR。

## 原子操作（兜底）

当 Pipeline IR 无法覆盖的极端场景，Agent 可以使用原子操作直接操作节点图：

| 操作 | 说明 |
|------|------|
| `create_node(type, params)` | 创建指定类型节点 |
| `remove_node(node_id)` | 删除节点 |
| `update_node(node_id, params)` | 更新节点参数 |
| `connect_nodes(from_id, from_slot, to_id, to_slot, to_input_name)` | 连接两个节点 |
| `disconnect_nodes(node_id, input_name)` | 断开连接 |
| `get_workflow()` | 获取当前工作流节点列表 |
