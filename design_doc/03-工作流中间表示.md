# 03 - 工作流中间表示（IR）

## 设计目标

屏蔽 ComfyUI 原生 JSON 的复杂度，提供一套简洁的中间表示（Intermediate Representation），让 Agent 能够用自然语言描述的工作流意图，被准确翻译为可执行的 ComfyUI 节点图。

```
自然语言 → Agent理解 → 中间表示(IR) → workflow_builder → ComfyUI原生JSON
```

## IR 设计原则

1. **扁平化**：对用户来说是一个配置字典，而非嵌套的节点图
2. **语义化**：字段名直观，对应的就是用户自然语言中的概念
3. **可扩展**：支持 `extra_params` 传递节点级高级参数
4. **可组合**：复杂工作流可以嵌套子工作流

## 核心 IR 结构

### 基础 IR（所有工作流共有）

```python
{
    "type": "txt2img",                    # 工作流类型
    "checkpoint": "dreamshaper_v8.safetensors",
    "positive_prompt": "...",
    "negative_prompt": "...",
    "width": 1024,
    "height": 1024,
    "seed": 42,
    "batch_size": 1,
    "steps": 20,
    "cfg": 7.0,
    "sampler_name": "euler_ancestral",
    "scheduler": "normal",
    "extra_nodes": {}                     # 预留：自定义额外节点
}
```

### 工作流类型定义

| type | 说明 | 特有字段 |
|------|------|----------|
| `txt2img` | 文生图 | - |
| `img2img` | 图生图 | `input_image`: 图片路径/base64, `denoise`: 0.0~1.0 |
| `inpaint` | 局部重绘 | `input_image`, `mask_image`, `denoise` |
| `upscale` | 超分放大 | `input_image`, `upscale_model`, `scale_factor` |
| `controlnet` | 控制网 | `input_image`, `controlnet_model`, `control_mode` |
| `ip_adapter` | IP适配器 | `input_image`, `ip_adapter_model`, `weight` |

> 注：初期先实现 `txt2img` 和 `img2img`，后续扩展。

### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `type` | string | 是 | - | 工作流类型 |
| `checkpoint` | string | 是 | - | 主模型文件名 |
| `positive_prompt` | string | 是 | - | 正向提示词 |
| `negative_prompt` | string | 否 | `""` | 反向提示词 |
| `width` | int | 否 | 512 | 图片宽度 |
| `height` | int | 否 | 512 | 图片高度 |
| `seed` | int | 否 | -1(随机) | 随机种子 |
| `batch_size` | int | 否 | 1 | 批量生成数量 |
| `steps` | int | 否 | 20 | 采样步数 |
| `cfg` | float | 否 | 7.0 | CFG scale |
| `sampler_name` | string | 否 | `"euler"` | 采样器名称 |
| `scheduler` | string | 否 | `"normal"` | 调度器 |
| `denoise` | float | 否 | 1.0 | 降噪强度(img2img/inpaint) |
| `input_image` | string | 否 | - | 输入图片路径或base64 |
| `mask_image` | string | 否 | - | 遮罩图片路径(inpaint) |
| `extra_nodes` | dict | 否 | `{}` | 自定义额外节点配置 |

## IR → ComfyUI JSON 翻译映射

以 `txt2img` 为例：

```
IR字段                  →  ComfyUI节点
─────────────────────────────────────────────────
checkpoint              →  LoadCheckpoint
positive_prompt          →  CLIPTextEncode (positive)
negative_prompt          →  CLIPTextEncode (negative)
width, height            →  EmptyLatentImage
steps, cfg, sampler,     →  KSampler
  scheduler, seed, denoise
(width, height)          →  VAEDecode (间接)
(batch_size)             →  SaveImage (间接)
```

### 生成的节点图结构

```
LoadCheckpoint ──► CLIPTextEncode(+) ──┐
                │                      ├──► KSampler ──► VAEDecode ──► SaveImage
                ├──► CLIPTextEncode(-) ─┘         │
                │                                  │
                └──► (VAE) ────────────────────────┘
                                                     
EmptyLatentImage ──────────────────────────┘
```

## 原子操作（兜底）

当中间表示无法覆盖的复杂场景，Agent 可以使用原子操作直接操作节点图：

| 操作 | 说明 |
|------|------|
| `create_node(type, params)` | 创建指定类型节点 |
| `remove_node(node_id)` | 删除节点 |
| `update_node(node_id, params)` | 更新节点参数 |
| `connect_nodes(from_id, from_slot, to_id, to_slot)` | 连接两个节点 |
| `disconnect_nodes(node_id, slot)` | 断开连接 |
| `get_workflow_graph()` | 获取当前工作流图结构 |

这些原子操作直接操作内存中的 ComfyUI 原生 JSON 图结构，最终通过 `save_workflow` 或 `execute_workflow` 生效。
